## Сборка

### Способ 1 — через vcpkg (рекомендуется, SFML скачается автоматически)

1. Установи [vcpkg](https://github.com/microsoft/vcpkg):
   ```
   git clone https://github.com/microsoft/vcpkg C:/vcpkg
   C:/vcpkg/bootstrap-vcpkg.bat
   ```
2. Скажи CMake где лежит vcpkg:
   ```
   cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake
   cmake --build build
   ```
   vcpkg сам прочитает `vcpkg.json` и скачает SFML 3.

### Способ 2 — вручную (скачать SFML самостоятельно)

1. Скачай **SFML 3.1.0** для своего компилятора с [sfml-dev.org](https://www.sfml-dev.org/download.php) и распакуй куда-нибудь, например `C:/SFML-3.1.0`
2. Укажи путь при конфигурации:
   ```
   cmake -S . -B build -DSFML_DIR="C:/SFML-3.1.0/lib/cmake/SFML"
   cmake --build build
   ```

### Требования
- **Компилятор:** MSVC 2022, MinGW-w64 или Clang с поддержкой C++17
- **CMake** 3.20+
- **Windows** (из-за `arial.ttf` из системных шрифтов)

После сборки рядом с `.exe` автоматически копируются `arial.ttf` и все SFML `.dll`.  
Тайлы карты (`tiles/`) и спрайты (`sprites/`) генерируются при первом запуске.

|---------|----------|

## Основная большая система — Danger Zones
**Файлы:** `Map.cpp`, `Game.cpp`

Ровно **1/4 всей карты** (75 тайлов из 300) случайно помечены как Danger Zones.  
Расположение меняется при каждом запуске и перезапуске.

Пока герой стоит на тёмно-бордовой клетке — он теряет **−3 HP за каждый ход**.  
Монстры намеренно спавнятся рядом с Danger Zones, создавая опасные зоны на карте.  
Сообщение `! DANGER ZONE ! -3 HP` появляется в combat log красным цветом.

Безопасная зона вокруг стартовой точки героя (1,1) гарантирует, что игра начинается без урона.

## Доп. механика 1 — Система усталости (Fatigue)
**Файл:** `Hero.cpp`

У героя шкала усталости **0–5**. Каждый шаг (и атака) увеличивает усталость.  
При максимальной усталости урон падает на **30%** (минимум 70% от базового).

**Отдых (Space):**
- Первое нажатие: сбрасывает усталость до 0. Монстры рядом атакуют в этот момент.
- Второе нажатие (усталость уже 0): тратит **10 маны** и восстанавливает **+8 HP**.

Мана не восстанавливается сама по себе — только из сундуков. Стартовый запас: **50 MP** (до 5 хилов).

## Доп. механика 2 — Сундуки с выбором награды (Loot Choice)
**Файл:** `Chest.cpp`, `Game.cpp`

На карте спавнится **2–4 сундука** в случайных проходимых клетках.  
При открытии появляется окно с **двумя кнопками** — выбор из двух наград разного типа (HP, ATK или Mana никогда не предлагаются оба одновременно).

| Тип | Эффект |
|-----|--------|
| `+HP` | Повышает **максимальный** HP (кап 120), и сразу добавляет столько же текущего HP |
| `+ATK` | Навсегда увеличивает базовый урон |
| `+Mana` | Восстанавливает ману для хилов на отдыхе |

## Архитектура

### Паттерн: Observer (EventManager)
`EventManager` (Singleton) позволяет системам общаться без прямых зависимостей.  
События: `monster_died`, `hero_exhausted`, `hero_rested`.  
Пример: монстр умирает → `notify("monster_died", name)` → UI пишет в лог.

### Абстрактный класс: Entity
Базовый класс для `Hero` и `Monster`.  
Чисто виртуальные: `draw()`, `getName()`.  
Виртуальные с переопределением: `takeDamage()`, `isAlive()`.  
Содержит `hp_cap` — потолок HP, специфичный для подкласса (Hero: 120, монстры: без ограничений).

### Принципы SOLID
- **S** — каждый класс имеет одну ответственность: `Map`, `CombatSystem`, `UI`, `EventManager`
- **O** — новые типы монстров добавляются без изменения существующего кода
- **L** — `Goblin` и `Troll` взаимозаменяемы как `Monster*`
- **I** — интерфейсы минимальны
- **D** — `Game` зависит от абстракций (`Entity*`, `Monster*`)

## Структура файлов
```
main.cpp              — точка входа
Game.h / .cpp         — главный игровой цикл, спавн монстров и сундуков
Map.h / .cpp          — карта, генерация Danger Zones (1/4 карты)
TileSet.h / .cpp      — пиксель-арт текстуры тайлов (16×16, сохраняются в tiles/)
SpriteSet.h / .cpp    — пиксель-арт спрайты сущностей (16×16, сохраняются в sprites/)
SpriteCache.h         — глобальный доступ к SpriteSet (singleton)
Entity.h              — абстрактный базовый класс с hp_cap
Hero.h / .cpp         — герой, усталость, механика хила за ману
Monster.h / .cpp      — Goblin и Troll
Chest.h / .cpp        — сундуки с выбором награды
CombatSystem.h / .cpp — расчёт боя (урон, усиленный удар, побег гоблина)
UI.h / .cpp           — HUD, список монстров, цветной combat log
EventManager.h        — паттерн Observer
FontCache.h           — единая загрузка шрифта (загружается один раз)
CMakeLists.txt        — сборка через CMake + SFML 3 (через vcpkg или вручную)
vcpkg.json            — зависимости для автоматической установки через vcpkg
```
